# ADR-0008 — Exclusion SysEx du MVP

- **Status:** Accepted
- **Date:** 2026-07-06
- **Supersedes:** none
- **Superseded by:** none
- **Liens :** PRD FR-8, N-2, NFR-20 ; spine AD-8 ; addendum A.1 (invariant 3)

> **Immutability :** Accepted ADRs are immutable. Future changes must create a new ADR that supersedes this one.

## Contexte

Les messages SysEx MIDI (`0xF0 … 0xF7`) sont : (1) à risque de sécurité (mise à jour firmware malveillante, per la spec W3C) ; (2) de taille **non bornée** (vecteur DoS) ; (3) **non pertinents** pour le MVP — l'envoi de patches DX7 SysEx et les presets par synthé sont explicitement hors MVP (N-2). Le MVP relaye uniquement 5 types channel-voice.

Une seule couche de filtrage (performer ou serveur) laisserait une faille si une couche est contournée ou ouverte. La défense en profondeur est exigée (FR-8).

## Décision

**Exclure SysEx du MVP par double défense.**

1. **Défense 1 (performer)** : filtre côté capture — tout message dont `event.data[0] === 0xF0` n'est **jamais** envoyé au serveur. SysEx silencieusement filtré, jamais affiché ni relayé.
2. **Défense 2 (serveur)** : `MidiEventSchema` (`@fmlw/shared`) n'expose **aucun** type SysEx → tout payload SysEx est rejeté automatiquement par la validation Zod.
3. **Navigateur** : `requestMIDIAccess({ sysex: false })` côté performer et listener.

## Conséquences

**Positives :**

- Risque firmware/DoS SysEx éliminé par design.
- Double défense : une couche défaillante ne compromet pas l'autre.
- Surface MVP réduite (pas de parsing SysEx non borné).

**Négatives :**

- Pas de patches DX7 SysEx / presets par synthé MVP — non-objectif assumé (N-2).
- Ajout futur de SysEx nécessitera : nouveau type au schéma + `requestMIDIAccess({ sysex: true })` + bornage taille. Post-traction, après rooms multiples.

## Alternatives considérées

- **SysEx autorisé avec bornage taille** : rejeté MVP — complexité (parsing non borné, gestion firmware) injustifiée ; N-2 l'exclut.
- **Filtrage serveur seul** : rejeté — violerait la défense en profondeur (FR-8) ; un performer compromis ou un client modifié pourrait tenter l'injection.
- **Filtrage performer seul** : rejeté — le serveur doit rester hostile aux payloads SysEx même si un client est modifié.
